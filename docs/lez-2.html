Git in 5 simple facts

  1. git lets you tell the story of your project

  2. git lets you travel time

  3. git helps you experiment

  4. git helps you back up your work

  5. git helps you collaborate

Fact 1 - the story

  • you use git to take snapshot of all the files in a folder

  • this folder is called repository or repo

  • when you want to take a snapshot of a file or files, you create a commit

  • when you commit a file or files, some information is saved along the changes to the file

    • who
    • when

  • you can add a brief & meaningful commit message to quickly tell what changes you’ve made

Saving copies vs making commits

Fact 2 - travel time

  • each commit is associated with an id called hash

  • you can move through your past commits

  • so I can tell git what commit I want to check out using the commit hash

Fact 3 - experiment

  • Projects are incremental, thing are non linear and sometimes you want to make easily discardable experiments

  • you can easily do it in git using branches

  • if you want to develop a new feature, or try a wild experiment working on a branch is saver

  • they can be easily thrown away

  • when you are done experimenting and you like your result you merge the branch

Fact 4 - back up

  • regular back up is a basilar best practice (e.g. Dropbox)

  • in git this place is called remote

  • The advantage of GitHub over GitLab or Bitbucket is that it has a larger network.

  • if you send your local changes to the remote you have made a push

  • if remote is ahead of your local directory, then you need a pull to get the changes

Fact 5 - collaboration

  • committing helps you tell others the evolution of your project

  • remotes provide exposure

  • git allows collaboration on the same project

Jargon

repository your project folder

commit a snapshot of your repo

hash an id for a commit

checkout moving through commit

branch a pointer to a commit

merge combining two branches

remote server with a repo on it

push send commits to a remote

pull get commits from a remote
